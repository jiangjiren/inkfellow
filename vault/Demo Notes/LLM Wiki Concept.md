# 什么是 LLM Wiki？

## 传统笔记的困境

你有没有这样的体验：

- 记了很多笔记，但需要的时候找不到
- 搜索关键词，结果一大堆，还是得手动翻
- 时间久了，很多写过的内容就像从未写过一样

这是因为传统笔记工具的使用方式是：

```
写 → 搜索 → 翻找 → 读
```

知识存进去了，但调用很费力。

## LLM Wiki 的解法

**LLM Wiki** 把流程变成：

```
写 → 直接问 AI
```

你的每一篇笔记，都成为 AI 的"记忆"。

### 举几个真实场景

**场景 1：找回旧想法**
> 「我三个月前记录过一个关于用户增长的判断，大概说的是什么？」

**场景 2：继续未完成的内容**
> 「帮我基于这篇读书笔记，整理出 5 个可以直接行动的要点。」

**场景 3：发现知识连接**
> 「这篇关于时间管理的笔记，和我其他关于专注力的内容有什么共同点？」

**场景 4：整理归纳**
> 「把我所有关于'产品设计'的笔记归纳成一个知识框架。」

## 为什么是"Wiki"？

Wiki 的本意是"可以随时编辑、互相链接的知识网络"。

LLM Wiki 在此基础上加了一层：**AI 作为知识网络的导航员**，帮你在自己的知识库里找路、建连接、发现盲区。

你是知识的创作者，AI 是你的知识助理。

## 和普通 AI 聊天有什么区别？

| | 普通 AI 聊天 | LLM Wiki |
|---|---|---|
| 知识来源 | AI 训练数据（通用知识） | 你自己写的笔记 |
| 回答风格 | 泛泛而谈 | 基于你的思考和语境 |
| 上下文 | 仅当前对话 | 整个笔记库 |
| 价值积累 | 对话结束即消失 | 笔记越多，越有价值 |
